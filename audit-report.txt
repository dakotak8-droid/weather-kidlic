PART 2: USER REVEAL WEATHER STORY EXECUTION TRACE
--------------------------------------------------------------------------------

When the user submits the form or clicks "Reveal the weather story" within the UI, the detailed end-to-end trace proceeds as follows:

1. Form Submit event is captured by `handleRevealStory(e)` inside `/src/components/BirthWeatherStory.tsx` (Line 938).
2. Input Validation:
   - Verifies the requested `birthDate` is present, parsed into `yearNum`, `monthNum`, `dayNum`, is not a future date, and is not prior to 1940.
3. City & Geocode Parsing:
   - Validates user input text. Resolves name, coordinates, country, and region using:
     a) Dropdown selection (`selectedCity`).
     b) Fallback lookup in Polish geographic registry maps.
     c) Fuzzy Geocoding Search: In the event of typed ambiguities, requests Open-Meteo Geocoding API (`https://geocoding-api.open-meteo.com/v1/search`), scores hits, filters results, and presents choices to select.
4. Historical Meteorological Archives Fetch:
   - Calls the public Open-Meteo Web API (`https://archive-api.open-meteo.com/v1/archive`) using coordinates and ISO date string formatting.
   - Extracts meteorological criteria (`weather_code`, max/min temperatures, precipitation, wind speed, and sunrise time).
5. Narrative API Generation:
   - Posts a JSON body containing parameters to `/api/generate-story`.
   - On Vercel (Production): Handled by `/api/generate-story.ts`.
   - On Node.js (Local Dev Container): Handled by the custom router in `/server.ts`.
6. Server-Side Execution Path:
   a) API Client check: Verifies `GEMINI_API_KEY`.
   b) Backups / Offline Paths:
      - If no key exists, or if API attempts fail or time out, executes `getOfflineBackupStory` with input arguments.
      - Injects period phrase structure using `applyTimeOfRecord` or neutralizes it via `makeStoryTimeNeutral`.
      - Appends the birth sentence: `" It was also the day a new little arrival entered the world."` / `" También fue el día en que una nueva y pequeña vida llegó al mundo."`
      - Returns status `200` JSON directly to client.
   c) Server-Side Gemini API Generation:
      - Formulates custom system instructions specifying exactly 90% weather descriptions, forcing strict third-person objective, restricting human references, and formatting structure rules.
      - Selects models via modern Google GenAI SDK and performs content generation using `gemini-3.5-flash`.
      - Checks responses against `validateGeneratedContent(story, quote, theme)`.
      - Rejects stories containing emotional pronouns or forbidden phrases (e.g. `holding you`, `our hearts`, etc.); performs up to 3 generation/validation iterations.
      - Neutralizes temporal elements if no valid `birthTime` parameter was provided.
      - Returns clean weather-only narrative (`theme`, `quote`, `story`) down the wire as REST response JSON.
7. Client-Side Parsing & Appending:
   - Captures the REST payload as `storyData`.
   - Append Sentence Injection:
     Inserts the birth indicator suffix to the AI-generated weather theme story, only if not already appended:
     `" It was also the day a new little arrival entered the world."` / `" También fue el día en que una nueva y pequeña vida llegó al mundo."`
8. Keepsake Card Construction:
   - Caches parameters onto state `revealResult`.
   - Paints city, weather metrics, theme tags, meteorological quote, and weather story directly to HTML5 Canvas.
   - Provides user interface controls to view and trigger a local client-side PNG/images download.
